# 🧠 费曼学习法日志：Chapter 4 - ReAct 范式

**日期**：2025-12-13

---

## 1. 概念定义 (Concept)
> **我想学的是什么？**
> ReAct (Reasoning + Acting) 模式，即“思考-行动-观察”的循环机制。

## 2. 模拟教学 (Teach it to a Child)
> **我的解释：**
> 为什么要记录 `Action` 和 `Observation`？
> 主要原因是：要让大模型知道上一步干了什么，根据前面来推导后面，之后再进行下一步，最终得到答案。
>
> 如果不给它看之前的记录，它就会像个健忘症患者，永远只记得最初的问题，而不知道自己已经查到了什么信息，陷入死循环或者瞎猜。

## 3. 回顾缺口 (Review Gaps)
> **刚才解释的过程中，哪里卡住了？**
> *   初次运行时遇到了 `serpapi` 库导入报错的问题，说明环境依赖和代码版本需要对应。
> *   理解 `history` 列表是如何被格式化成字符串并插入到 `prompt` 中的。

## 4. 简化升华 (Simplify & Refine)
> **我的比喻：**
> **做菜（Cooking）**
> *   **Thought (思考)**：看菜谱，想下一步该切菜还是下锅。
> *   **Action (行动)**：拿起刀切菜，或者打开煤气灶。
> *   **Observation (观察)**：看到菜切好了，或者水开了。
> *   **History (历史)**：这就像是做菜的记忆。如果没有记忆，你可能切完菜转头就忘了，又去切了一遍，永远做不出一道完整的菜。最终这些不同的步骤组装起来，就成了一道菜（最终答案）。

## 🤖 AI 助教反馈区
**点评**：
你的“做菜”比喻非常精准！
*   ReAct 的核心确实就是**“短期记忆维护”**。
*   LLM 本身是无状态的（Stateless），每一次请求都是全新的。
*   `history` 变量充当了它的**外部存储器**（External Memory），把之前的“切菜结果”（Observation）喂回给它，它才能决定“下一步该炒菜了”。

**评分**：⭐⭐⭐⭐⭐ (完全掌握核心逻辑)
